## ⚠️ 중요: 데이터 및 환자 정보 관리 지침

### **1. 데이터 및 환자 정보 업로드 금지**
- 민감한 **데이터** 또는 **환자 정보**는 절대 GitHub에 업로드하지 마세요.
- 개인 식별 가능 정보(PII) 및 의료 데이터는 철저히 보안 정책을 준수하여 관리해야 합니다.
- 필히 올려야할 경우, 비식별화 후 소수의 데이터만 업로드 

### **2. .gitignore 설정**
Git을 사용하여 데이터를 관리할 때, 민감한 파일이 커밋되지 않도록 `.gitignore`를 설정하세요.  
다음 예시를 참고하여 `.gitignore` 파일에 아래 내용을 추가하세요:

```bash
# .gitignore 예시
# 데이터 및 민감 정보 관련 파일
*.csv
*.xlsx
*.dcm
*.json
*.zip
/data/

```

## ❔ GitHub 사용법
### 1. Repository 생성
- **New Repository**로 새로운 원격 저장소 생성
- **Repository name**: 저장소명을 지정(띄어쓰기 대신 _를 사용)
- **Description**: README 파일의 저장소명 아래의 설명 작성
- **Choose Visibility**: Public/Private 중 선택
- **Add README**: Repository의 표지와 같은 페이지 추가 여부 선택
- **Add .gitignore**: 폴더나 파일을 올릴 때, Github에 보여주고 싶지 않은 파일을 설정
	- (직접 작성도 가능하지만 기본적으로 언어별로 제공해주고 있으니 찾아서 추가 가능)



### 2. Repository 내 컴퓨터에 복제
！GitHub는 원격 저장소이므로, Clone(복제)를 통해서 내 로컬 저장소로 가져오는 과정 </br>
- 현재 Github의 주소 복사
- Terminal에서 이 저장소를 저장하고 싶은 위치로 이동
	- (예시 그림에서는 Desktop에 저장)
- 배경화면에 해당 저장소 이름의 폴더가 생성됨



### 3. 폴더 및 파일 Github Repository에 올리기
！원격과 로컬 저장소가 Git을 통해 연결되어 있기 때문에, 내 로컬 저장소에서의 변경한 것들을 원격 저장소에 동일하게 저장할 수 있음 </br>
- 생성된 로컬 저장소에 Github에 올리고자 하는 폴더 및 파일을 옮기기
	- 올리게 되면, 현재 로컬과 원격 저장소 간의 차이가 발생함
- Terminal로 해당 로컬 저장소 위치로 이동
- **`git status`** 로 현재 어떤 변경이 있었는지 확인
	- test_images 폴더, test_upload.py 가 추적하지 않은 파일로 뜨는 것을 확인
- **`git add .`** 으로 변경 사항들 add 하기
	- .은 전체라는 뜻으로 지정해서 추가도 가능
- **`git commit –m “메세지내용”`** 으로 commit 하기(실제로 추가 하겠다는 확신 메세지?로 생각하면 편함)
- **`git push origin main`** 으로 실제 원격 저장소에 push 하기
